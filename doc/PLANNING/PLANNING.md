# 連絡帳アプリ仕様

## 1.1. 要件定義の確認と詳細化

- [ ] **必須ゴール・＋ α ゴールの再確認:**
  - [ ] 各学年、各クラス毎に本システムが利用できる
  - [ ] 現状の連絡帳に記載されている内容を生徒毎に登録できる
    - [ ] 現在の連絡帳の記載内容
      - [ ] 日付
      - [ ] 体調：10段階評価（1:悪い～5:普通～10:絶好調）
      - [ ] メンタル：10段階評価（1:悪い～5:普通～10:絶好調）
      - [ ] 今日の振り返り：最大1000文字
      - [ ] スタンプ（担任が押す）
    - [ ] 生徒は提出日の前登校日の内容を登録する
      - [ ] 土日祝日・長期休暇は考慮しない
  - [ ] 担任は生徒の記録を確認し、既読処理できる
    - [ ] 担任は生徒の記録を編集することはできない
    - [ ] （＋α）「いいね」などのスタンプ機能
    - [ ] （＋α）時系列での各生徒の状態確認
    - [ ] （＋α）全体感の把握
  - [ ] 生徒は自身の過去記録を閲覧することができる
    - [ ] 生徒は過去記録を改変できない
    - [ ] 既読処理が行われた時点で「過去記録」にする
  - [ ] 担任は1日の生徒全体の提出状況を把握することができる
  - [ ] 担任は過去の記録を閲覧できる
  - [ ] 担任は連絡帳の内容についてのログを残すことができる
    - [ ] ログは生徒からは閲覧できない
  - [ ] （＋α）学年主任は各クラスの状況を把握することができる
  - [ ] ユーザー管理者は生徒・担任の作成、各ユーザーへのクラス・学年の割り当てを行うことができる
    - [ ] （＋α）教師への各役職の割り当て（副担任・学年主任等）
    - [ ] （＋α）生徒への委員会・部活等の所属情報の割り当て
  - [ ] （＋α）ユーザー管理者は学校カレンダーを登録できる
  - [ ] PC上での上記要件動作の確認
    - [ ] （＋α）レスポンシブ対応
  - [ ] パスワードリセット機能
  - [ ] （＋α）年度更新機能
- [ ] **機能要件の詳細化:**
  - [ ] **生徒機能:**
    - [ ] 入力項目定義
      - [ ] 日付[date]
      - [ ] 体調[int](1-10)
      - [ ] メンタル[int](1-10)
      - [ ] 今日の振り返り[varchar(1000)]
    - [ ] 各担任宛に確認フロー発行
      - [ ] 各担任ページに通知が表示される
      - [ ] （＋α）登録したアドレスにメールが送信される
        - [ ] デフォルトはオフ
      - [ ] 自身の記録の閲覧画面詳細
        - [ ] 表（デフォルト）
          - [ ] 日付
          - [ ] 体調
          - [ ] メンタル
          - [ ] 今日の振り返り
            - [ ] 吹き出しボタンを配置しておいて、クリックすると詳細モーダルを表示する
            - [ ] 詳細モーダルは前日、翌日に移動できるボタンを付ける
        - [ ] （＋α）一時保存機能
        - [ ] （＋α）グラフ機能
  - [ ] **担任機能**
    - [ ] 生徒一覧画面
      - [ ] 生徒氏名
      - [ ] 提出状況
        - [ ] 記録テーブルに該当生徒・該当日付のレコードが存在するか否かで判定する
      - [ ] 詳細ボタン
        - [ ] クリックするとその日の記載内容が見られる
        - [ ] 詳細画面内容
          - [ ] 既読/キャンセルボタン
          - [ ] メモ欄
            - [ ] 保存ボタン（ボタンを押すとその都度上書き）
      - [ ] 各生徒の時系列表示
      - [ ] （＋α）グラフ機能
      - [ ] （＋α）各担任、学年主任へのシェア機能
  - [ ] **ユーザー管理者機能**
    - [ ] ロール定義
      - [ ] 生徒
      - [ ] 担任
      - [ ] ユーザー管理者
      - [ ] （＋α）副担任
      - [ ] （＋α）学年主任
  - [ ] **（＋α）学年主任機能**
    - [ ] 各クラスの状況閲覧・サーベイ機能
    - [ ] （＋α）各担任・副担任への共有機能
