## プロジェクトの概要

2025年10月頃に、架空の中学校での連絡帳をシステム化する、という想定で制作した成果物です。
作業期間は4週間、100時間以内という制限があり、提示された要求に従い、要件定義・設計・開発を行いました。

## 今後の展望・改善点

本システムの運用と発展の面から、以下の改善点や新機能の実装を検討しています。

### 認証・セキュリティ

- 現在UI上からは行えない、ユーザー自身によるパスワードリセット機能を実装する。
- 現在の最小文字数（6文字）制限に加え、より厳格なバリデーションの適用や、ソーシャルログインの導入を検討する。

### 全体的なパフォーマンスとUXの向上

- スマートフォンでの閲覧時（特にテーブル表示）に視認性の課題があるため、より高度なレスポンシブ対応を行う。
- 現状の1度限りのデータ取得から、Firestoreのリアルタイムリスナーへ移行。生徒の提出や「気になるフラグ」の変更などのステータス変化を動的に検知し、都度リロードを行う手間を省く。
- 教師ダッシュボードでのクラス平均スコアや提出率の算定を、クライアント側での全件読み込みからバックグラウンド処理へ移行。結果を別コレクションに保存することで、パフォーマンスを向上させる。
- 過去記録の検索性向上のため、検索・ソート・絞り込み機能を追加し、ページネーションを実装する。
- 生徒アカウントと紐づく保護者権限を設け、子供の記録閲覧（教員メモは非表示）や教師との連絡機能を実装することで、システムの利用価値をさらに高めたい。

### 権限別の機能拡張

#### 管理者

- 現在の新規ユーザー作成機能に加え、進級・卒業・異動等に伴うステータス変更やアカウント削除機能を実装し、より実践的なシステムにする。
- 年度替わりなど入力件数が大幅に増える時期を想定し、ExcelやCSVによる一括処理機能を導入する。

#### 生徒

- 生徒自身のセルフマネジメントを促すため、自身のコンディション推移を客観的に把握できるグラフ表示機能を追加する。
- 提出忘れを防ぐため、特定時刻以降の画面アラートやメール通知機能を実装する。
- テキスト入力や言語化が苦手な生徒への配慮として、気分を表すスタンプ（「楽しかった」「普通」「疲れた」等）や定型文ボタンを導入する。
  - ※工夫点と課題: 「普通」スタンプをスコア5点に換算するなど数値データと連動させる仕組みを想定。ただし、定型文での提出が常態化する懸念もあるため、実際の学校の方針とすり合わせた上での導入が必要。

#### 担任

- 「既読にする」ボタンの代わりに「いいね」「がんばったね」などのリアクションスタンプや、簡単なフィードバックコメントを残せる機能を追加する。
  - ※課題: 教員と生徒間のDM機能のように運用されるとトラブルの要因になり得るため、権限や文字数・内容制限などの設計は慎重な検討を要する。
- 「メンタルスコアが3日連続で4以下」「2日連続未提出」などの閾値を設け、該当生徒を赤くハイライト表示したりメール通知を行う仕組みを導入する。  
  - これを発展させ、AI連携により入力テキストやスコア推移から注意を要する生徒の兆候を事前検知し、アラートを出す機能も視野に入れている。
- ダッシュボードに前日データだけでなく、直近30日間や指定期間など、任意の期間でのクラスサマリーを表示可能にする。
- クラスの枠を越えた連携のため、新たに「気になる生徒」の学年共通ダッシュボードを設け、他クラスの担任もコメントを残せる仕組みを構築する。
- 生徒一覧画面で「気になる」フラグが付いた生徒をひと目で判別できるよう、ハイライト表示を追加する。

#### 学年主任

- 現在の「気になる」生徒の個別状況把握に加え、学年全体やクラス別の状況・サマリーをグラフ等で俯瞰できるダッシュボードを実装する。
- 担任の個別メモとは別に、学年全体やクラス単位の運用に関して主任が書き込めるメモ機能を追加する。
